# 投影市場事件跑馬燈設計

日期：2026-06-21

## 目標

主持人啟用新事件時，投影先播放一次新聞號外報紙動畫；動畫結束後，事件內容持續顯示於投影頁頂部的市場事件區，以水平跑馬燈循環播放。

## 顯示內容

每個啟用事件依序顯示：

1. 事件名稱
2. 新聞描述
3. 實際漲跌影響

例如事件一：

`事件一：晨霧退散，資金湧入金域 ｜ 極光金域上漲，晨霧棲城下跌。 ｜ 極光金域 ▲25% · 晨霧棲城 ▼10% · 金融、商業、投資、商貿、展覽 ▲10%`

事件內容循環捲動。多個事件同時啟用時，按照事件編號排列並接續輪播。

## 實質影響

- 區域漲跌由 `EVENTS[index].regionMult` 計算。
- 資產類型漲跌由 `EVENTS[index].typeMult` 計算，相同倍率合併顯示。
- 事件四額外顯示主持人指定區域下跌 15%。
- 未知事件或無效懲罰區域不顯示。

## 視覺與動態

- 沿用投影 Header 既有的青色市場事件橫幅。
- 左側保留廣播圖示與「市場事件」標籤。
- 右側文字區使用遮罩裁切，內容由右向左持續捲動。
- 為了無縫循環，畫面會渲染兩份相同事件內容。
- 使用者偏好減少動態時停止位移，仍完整顯示事件文字。
- 沒有啟用事件時，不顯示跑馬燈橫幅。

## 與其他動畫的關係

新事件會進入一次性動畫佇列，優先序為事件、樂透、拍賣。報紙只播放一次，頂部跑馬燈則在事件啟用期間持續顯示；常駐拍賣舞台維持原有行為。

## 測試

- 單一事件包含名稱、新聞及所有實際影響。
- 多事件依事件編號排序。
- 事件四包含主持人指定區域下跌 15%。
- 未知事件被忽略。
- 投影頁不再渲染事件全螢幕 dialog。
- 瀏覽器確認跑馬燈持續捲動，原投影 UI 未被遮擋，console 無錯誤。
