# 投影市場事件號外動畫設計

日期：2026-06-21

## 目標

主持人在控制台觸發市場事件後，投影頁不修改原有常駐 UI，而是在最上層播放一次事件動畫。動畫先讓一張「新聞號外」從遠處旋轉飛入，撞擊畫面中央後展開成事件彈窗，清楚顯示事件名稱、新聞描述與實質影響。

事件彈窗展開後停留 30 秒，再以 0.7 秒淡出。事件只播放一次，不提供手動重播。

## 觸發規則

- 投影頁第一次載入時，只記錄目前已啟用的事件，不補播歷史事件。
- 後續每次取得快照時，比對前後 `activeEvents`，只為新加入的事件建立動畫。
- 關閉事件不顯示解除通知，也不播放任何動畫。
- 若同一次快照新增多個事件，依事件編號由小到大排隊播放。
- 若事件曾被關閉後再次啟用，會視為一次新的觸發並再次播放；正常遊戲流程預期事件只會持續疊加。

## 視覺設計

### 前導動畫

1. 投影原有畫面逐步壓暗，但保持可辨識。
2. 一張縮小、略微傾斜的報紙從畫面遠處旋轉飛入。
3. 報紙抵達中央時短暫撞擊、放大並產生輕微震動與光暈。
4. 報紙展開成完整「新聞號外」彈窗。

前導動畫約 1.5 至 2 秒，動畫結束後才開始計算 30 秒停留時間。

### 新聞號外彈窗

彈窗採用已確認的 C 方案：

- 米白色報紙底、深棕色文字與紅褐色報頭。
- 報頭顯示 `CAPITAL WAR DAILY`。
- 顯示「市場號外」與事件編號。
- 事件名稱為最大標題。
- 顯示 `EVENTS[index].news` 作為新聞摘要。
- 「市場變化」區塊以條列方式呈現實質影響。
- 上漲使用綠色數值與向上符號；下跌使用紅色數值與向下符號。
- 彈窗保持適合投影遠距閱讀的大字級與高對比。

## 實質影響內容

內容直接由 `EVENTS` 的規則資料產生，避免畫面文字與實際計算規則分離。

- `regionMult` 轉換為區域漲跌百分比。
- `typeMult` 將相同倍率的資產類型合併顯示。
- 倍率大於 1 顯示上漲；小於 1 顯示下跌。
- 事件四額外讀取 `event4Penalty`，顯示主持人選定區域再下跌 15%。
- 未受影響的區域或類型不列出。

例如事件一會顯示：

- 極光金域：上漲 25%
- 金融、商業、投資、商貿、展覽：上漲 10%
- 晨霧棲城：下跌 10%

## 動畫優先順序與佇列

投影動畫使用單一前端佇列，避免事件、樂透與拍賣動畫互相覆蓋或遺失：

1. 市場事件
2. 大樂透
3. 拍賣加價或成交

常駐拍賣舞台不屬於一次性動畫，仍維持在最底層。一次只播放一個全螢幕覆蓋動畫；目前動畫結束後，才從佇列取出下一個項目。

若低優先動畫已經開始播放，不中途打斷；新事件會插入等待佇列中所有尚未播放項目的前方，於目前動畫結束後優先播放。

## 元件與責任

### 事件差異偵測器

接收前後兩次 `activeEvents`，輸出新加入的事件編號。它不依賴 React，可用單元測試覆蓋首次載入、單事件、多事件與事件移除等情況。

### 事件影響格式化器

接收事件定義與 `event4Penalty`，輸出供 UI 顯示的影響列。百分比由倍率計算，不另寫一套固定文案。

### 投影動畫協調器

統一接收事件、樂透與拍賣的動畫訊號，維護等待佇列及目前播放項目。依優先順序排入，但不打斷已開始的動畫。

### EventNewspaperOverlay

負責報紙飛入、撞擊展開、30 秒停留與 0.7 秒淡出。元件只接收已格式化的事件資料與完成 callback，不讀取快照或自行決定優先順序。

## 資料流程

1. 主持人觸發事件，後端更新 `GameState.activeEvents`。
2. 投影頁定期取得公開快照。
3. 事件差異偵測器找出新增事件。
4. 將事件名稱、新聞與格式化影響建立成動畫項目。
5. 投影動畫協調器依優先順序排入佇列。
6. 輪到事件時顯示 `EventNewspaperOverlay`。
7. 前導動畫完成後停留 30 秒，再淡出並通知協調器播放下一項。

## 邊界情況

- 快照輪詢重複回傳相同事件時，不重播。
- 投影頁重新整理時，不補播已啟用事件。
- 事件資料不存在時忽略該事件，不讓投影頁崩潰。
- 事件四未提供有效懲罰區域時，只顯示其一般影響，不顯示額外 15% 條目。
- 事件顯示期間發生樂透或拍賣動畫時，動畫保留在佇列，事件結束後依序播放。
- 多個拍賣加價在長事件彈窗期間發生時，只保留仍具意義的最新加價；成交動畫不得被後續加價取代。

## 測試與驗證

### 單元測試

- 首次載入不產生事件動畫。
- 新增一個事件只產生一次。
- 同時新增多個事件時依編號排序。
- 移除事件不產生動畫。
- 各事件倍率正確轉換為百分比與漲跌方向。
- 事件四正確顯示選定區域下跌 15%。
- 佇列遵守事件、樂透、拍賣優先順序。
- 新事件不會中斷正在播放的動畫。

### 瀏覽器驗證

- 主持人觸發事件後，投影出現報紙飛入動畫。
- 動畫展開後顯示正確事件名稱、新聞與實質影響。
- 彈窗展開後維持約 30 秒，再平滑淡出。
- 投影原有 UI 未被修改，只在背景被壓暗。
- 事件播放期間觸發樂透或拍賣，後續動畫沒有遺失。
- 瀏覽器 console 無 React key、effect cleanup 或 hydration 警告。
